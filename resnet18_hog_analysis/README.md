# Emotion Recognition and Frame Selection Evaluation Pipeline

This repository contains a suite of Python scripts designed for processing video datasets (specifically `devemo` and
`devemo+`), extracting representative frames, and evaluating machine learning models on emotion recognition tasks. The
pipeline leverages deep learning (PyTorch, TensorFlow) and classic computer vision techniques (MTCNN, HOG) to assess the
effectiveness of various frame-selection strategies.

## File Overview

The repository consists of four main scripts:

### 1. `binary_prepare_dataset.py`

This script processes raw video files to create 2D image summaries for **binary classification** (Negative vs. Other).

* **Frame Selection**: Extracts frames using 9 distinct strategies (e.g., `uniform`, `random`, `edges`, `sharpest`,
  `hog_diff`, `resnet_kmeans`).
* **Temporal Chromatic Fusion**: Combines multiple selected frames (3, 4, or 5) into a single composite image using
  colored temporal tints and alphas.
* **Timeline Visualization**: Overlays a visual timeline onto the bottom of the composite image to show exactly when the
  frames were extracted.
* **Outputs**: Saves the generated images into the `output/` directory.

### 2. `multiple_strategies_prepare_dataset.py`

This script mirrors the functionality of the binary preparation script but maps the dataset labels into **multiple
classes** (e.g., Confusion, Anger, Disgust, Surprise, Happiness, Neutral) instead of binary categories. It utilizes the
exact same temporal chromatic fusion and frame-selection strategies to prepare data for multiclass training.

### 3. `HogModel.py`

A standalone TensorFlow/Keras pipeline that trains a custom Dense Neural Network on Histogram of Oriented Gradients (
HOG) features.

* **Face Detection**: Uses `MTCNN` (via `facenet_pytorch`) to detect and crop faces from sequences of 8 frames per
  video.
* **Feature Extraction**: Computes spatial HOG features for the cropped faces and concatenates them into a single vector
  representation per video.
* **Training Pipeline**: Handles class imbalances via custom oversampling and class weights. Saves the best model,
  accuracy/loss plots, and a confusion matrix to the `hog_output/` directory.

### 4. `multiple_mean_evaluate_strategies.py`

A PyTorch evaluation script designed to benchmark the composite images generated by the dataset preparation scripts.

* **Model**: Fine-tunes a ResNet18 Convolutional Neural Network on the generated 2D images.
* **Robust Evaluation**: Runs multiple training loops (default: 10 runs) across all frame selection strategies to
  calculate the mean test accuracy and standard deviation.
* **Visualizations**: Generates Grad-CAM (Gradient-weighted Class Activation Mapping) images, combined mean confusion
  matrices, and test prediction examples.
* **Outputs**: Saves all plots and a final CSV summary (`final_summary_matrix_multiple_classes.csv`) in the `results/`
  directory.

## Setup and Dependencies

To run the scripts, ensure you have the following directories at the root level containing your `.mp4` video files:

* `devemo/`
* `devemo+/` (Should include a `devemo+.json` file containing the labels).

### Required Libraries

You will need a Python environment with the following packages installed:

* `opencv-python` (`cv2`)
* `numpy`, `pandas`
* `Pillow` (`PIL`)
* `scikit-learn`, `scikit-image`
* `torch`, `torchvision`, `facenet-pytorch`
* `tensorflow`
* `matplotlib`, `seaborn`
* `grad-cam` (`pytorch_grad_cam`)

## Usage Pipeline

**Step 1: Prepare the Dataset**
Run one of the preparation scripts to generate the composite frame images.
For binary classification:

```bash
python binary_prepare_dataset.py
```

For multiclass classification:

```bash
python multiple_strategies_prepare_dataset.py
```

*(Images will be saved to `output/`)*

**Step 2: Evaluate the Strategies**
Once the `output/` folder is populated, run the evaluation script to train ResNet18 models on the generated images and
generate reports.

```bash
python multiple_mean_evaluate_strategies.py
```

*(Results, plots, and GradCAM visualizations will be saved to `results/`)*

**Step 3: Train the HOG Sequence Model (Alternative Approach)**
To train the sequential HOG-based model directly on the raw videos:

```bash
python HogModel.py
```

*(Training history and confusion matrices will be saved to `hog_output/`)*
